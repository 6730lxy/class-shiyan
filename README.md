# ex04
完成基于触摸传感器的“自锁”开关
关联基础：实验2 (基础IO)、实验4 (触摸引脚)
提示：基础的触摸实验通常是“摸着亮，松开灭”，要求引入布尔型状态变量（如 `bool ledState`）和边缘检测逻辑（判断上一次未触摸，当前被触摸的瞬间）。
只有在检测到按下瞬间，才翻转LED的状态。
为了防止手抖导致的误触发，还需要加入软件防抖逻辑。
预期实验结果：摸一下触摸引脚，LED亮起并保持长亮；松开手再摸一下，LED熄灭

# ex05
完成多档位触摸调速呼吸灯
关联基础：实验3 (PWM呼吸灯)、实验4 (触摸引脚)
提示：结合呼吸灯的PWM代码和触摸自锁开关逻辑。定义一个变量表示“速度档位”（如 1、2、3档），每次触发触摸引脚，档位循环切换（1->2->3->1...）。
在PWM循环中，根据当前的档位变量，改变 `delay()` 的时长或占空比每次递增的步长（Step）。
预期实验结果：LED持续呈呼吸灯效果。
用户每触摸一次引脚，LED“呼吸”的节奏就会发生改变（如从缓慢呼吸变为急促呼吸，共有三个明显的速度级别）。

# ex06
完成警车双闪灯效（双通道PWM）
关联基础：实验3 (PWM呼吸灯)
提示：要求在面包板上连接两个LED灯至不同的引脚，初始化两个独立的PWM通道。
编写逻辑让两个灯的亮度呈现“反相”关系：当灯A的占空比从0增加到255时，灯B的占空比恰好从255减小到0。
预期实验结果： 两个LED灯呈现平滑交替的渐变闪烁效果，一个变亮的同时另一个逐渐变暗，过渡非常柔和。

# ex07
完成Web网页端无极调光器
参考PWM呼吸灯和Web服务器
提示：要求修改HTML代码，在网页中加入一个滑动条（`<input type="range" min="0" max="255">`）。通过 JavaScript 监听滑动条的变动，并通过 `fetch()` 或 `GET` 请求将滑动条的具体数值发送给ESP32。ESP32解析URL中的数值并赋值给PWM通道。
预期实验结果：手机或电脑浏览器打开ESP32网页，页面上出现一个滑动条。拖动滑动条时，开发板上的LED亮度会随着滑动条的位置实时、平滑地改变。

# ex08
完成物联网安防报警器模拟实验
参考基础IO、触摸引脚、Web服务器
提示：将ESP32作为一个“安防主机”。网页端提供两个按钮：“布防(Arm)”和“撤防(Disarm)”。使用全局变量记录系统状态。
在布防状态下，如果触碰引脚，则LED开始狂闪（报警状态），即使手松开，LED闪烁也不会停止，直到用户在网页端点击“撤防”。
预期实验结果： 未布防时触摸引脚无反应；在网页点击布防后，一旦有人触碰引脚，LED开始高频闪烁锁定报警；只有通过网页点击撤防，LED才会熄灭并重置状态。

# ex09
完成实时传感器Web仪表盘
提示：不要只做控制（下发），也要做数据采集（上报）。
要求在ESP32的Web Server中不仅下发HTML页面，还要能实时返回触摸传感器的模拟量数值。网页端使用AJAX技术不断拉取该数值，并显示在网页中央。
预期实验结果： 网页上显示着一个实时跳动的数字。当手逐渐靠近ESP32引脚时，网页上的数字实时变小；手指离开时，数值恢复。实现类似仪器仪表的数据实时监控面板。
